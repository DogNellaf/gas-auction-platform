# Газовый аукцион

> [🇬🇧 English](README.md) | 🇷🇺 Русский

Веб-платформа для B2B-торгов газом. Компании регистрируются, проходят одобрение администратора, участвуют в аукционах и подают ценовые предложения. Побеждает наибольшая ставка.

## Возможности

- Публичный список аукционов — просмотр открытых аукционов с текущими ценами и сроками окончания
- Регистрация компании — регистрация с названием компании, организационно-правовой формой и контактными данными; аккаунт активируется после одобрения администратором
- Подача ставок — авторизованные пользователи подают ценовые предложения; каждая ставка должна превышать текущую цену не менее чем на настроенный процент шага
- Личный кабинет — просмотр поданных заявок и их результатов; редактирование профиля
- Панель администратора
  - Создание, редактирование и управление аукционами (статус, стартовая цена, время окончания, шаг цены)
  - Ручное завершение любого аукциона; победитель определяется автоматически по наибольшей ставке
  - Просмотр всех заявок и деталей договора для победителей
  - Одобрение и блокировка пользователей
- Автоматическое завершение аукционов — планировщик завершает просроченные открытые аукционы каждую минуту и назначает статусы «Победа» / «Проигрыш»

## Установка

```bash
# Клонировать репозиторий
git clone <repository-url>
cd Arbitration

# Установить PHP-зависимости
composer install

# Установить фронтенд-зависимости
npm install

# Скопировать конфиг окружения и настроить базу данных
cp .env.example .env
# Отредактировать .env: задать DB_DATABASE, DB_USERNAME, DB_PASSWORD

# Сгенерировать ключ приложения
php artisan key:generate

# Выполнить миграции и заполнить начальными данными
php artisan migrate --seed

# Собрать фронтенд-ресурсы
npm run build

# Запустить сервер разработки
php artisan serve
```

Приложение будет доступно по адресу `http://127.0.0.1:8000/`.

Учётные данные после seed:

| Роль          | Email             | Пароль   |
|---------------|-------------------|----------|
| Администратор | admin@example.com | password |
| Пользователь  | user@example.com  | password |

## Переменные окружения

Для продакшн-развёртывания задайте эти переменные вместо значений по умолчанию:

| Переменная      | Описание                                   | По умолчанию       |
|-----------------|--------------------------------------------|--------------------|
| `APP_KEY`       | Ключ шифрования приложения                 | _(генерируется)_   |
| `APP_ENV`       | Окружение (`local`/`production`)           | `local`            |
| `APP_DEBUG`     | Режим отладки (`true`/`false`)             | `true`             |
| `APP_URL`       | Базовый URL приложения                     | `http://localhost` |
| `DB_CONNECTION` | Драйвер базы данных                        | `mysql`            |
| `DB_DATABASE`   | Имя базы данных                            | —                  |
| `DB_USERNAME`   | Имя пользователя БД                        | —                  |
| `DB_PASSWORD`   | Пароль пользователя БД                     | —                  |

Для автоматического завершения аукционов в продакшне добавьте cron-задачу:

```
* * * * * cd /path-to-project && php artisan schedule:run >> /dev/null 2>&1
```

## Запуск тестов

```bash
php artisan test
```

Тесты используют базу данных SQLite в памяти и не затрагивают данные разработки.

## Структура проекта

```
app/
├── Enums/            # AuctionStatus, BidStatus
├── Http/
│   ├── Controllers/  # Контроллеры
│   └── Middleware/   # AdminMiddleware
├── Models/           # Auction, Bid, User, LegalForm
database/
├── factories/        # Фабрики моделей для тестов
├── migrations/       # Схема базы данных
└── seeders/          # Начальные данные (ОПФ, демо-аккаунты)
resources/views/
├── admin/            # Шаблоны панели администратора
├── auth/             # Вход и регистрация
├── home/             # Личный кабинет
└── layouts/          # Базовые макеты
routes/
├── web.php           # HTTP-маршруты
└── console.php       # Планировщик задач
tests/
├── Feature/          # HTTP / контроллерные тесты
└── Unit/             # Юнит-тесты моделей
```
